AI Testing

Testing the untestable.

Traditional QA breaks when the system is non-deterministic. Here's how I approach testing AI products.

01

Non-Deterministic Output Testing

When an LLM gives a different answer every time, you test the shape of the output — not the exact text. Schema validation, tone checks, length constraints, and semantic similarity scoring replace exact-match assertions.

02

RAG Pipeline Validation

Retrieval-Augmented Generation pipelines need checks at every stage: retrieval quality, context relevance, faithfulness of the final answer to source documents. RAGAS metrics like faithfulness and answer relevance provide measurable quality signals.

03

Prompt Injection Testing

AI systems have a new class of security vulnerability. Testing includes injecting adversarial prompts, attempting role-play overrides, and checking whether the system leaks system prompt instructions or bypasses guardrails.

04

Hallucination Detection

Validating that an AI system doesn't confidently state false facts. Approaches include ground-truth comparison, cross-reference checks, and confidence calibration testing against known datasets.
